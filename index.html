Інформаційна система
оперативного контролю авіаційного двигуна

Information System for Operational Control of Aircraft Engine

Система моделює динаміку системи автоматичного регулювання (САР) авіаційного двигуна методом чисельного інтегрування диференціальних рівнянь. Реалізовано на C++ з використанням методу Рунге-Кутта 4-го порядку.

// 01 — актуальність

Чому це
важливо?

Сучасні авіаційні двигуни є складними об'єктами керування, де навіть незначне відхилення від розрахункового режиму роботи може призвести до катастрофічних наслідків.

Існуючі системи контролю часто не відповідають вимогам щодо швидкодії та точності. Дана робота спрямована на розробку алгоритму та програми для оперативного виявлення відхилень режиму роботи САР від розрахованого.

Вимоги: точність не менше 95%, час реакції не більше 1 секунди.

95%
Мінімальна точність виявлення відхилень
<1с
Максимальний час реакції системи
4-й
Порядок диференціального рівняння двигуна
24
Unit-тести (Google Test)

// 02 — мета

Мета та
завдання

Мета дослідження

Розробка алгоритму та програми для оперативного виявлення відхилення режиму роботи системи автоматичного регулювання авіаційного двигуна від розрахованого з точністю не менше 95% та часом реакції не більше 1 секунди.

_

ОСНОВНІ ЗАВДАННЯ

  • 01
    Дослідити математичну модель авіаційного двигуна як об'єкт керування
  • 02
    Реалізувати метод Рунге-Кутта 4-го порядку для чисельного розв'язання ОДУ
  • 03
    Розробити алгоритм виявлення відхилення режиму роботи від розрахованого
  • 04
    Забезпечити експорт результатів у CSV та автоматичну візуалізацію (Python/matplotlib)
  • 05
    Провести верифікацію системи (24 unit-тести, Google Test Framework)

// 03 — методологія

Методологія
дослідження

∫

МЕТОД РУНГЕ-КУТТА

Чисельне інтегрування диференціального рівняння 4-го порядку методом RK4. Обчислення та збереження всіх похідних до 4-го порядку включно.

⚙

МОДЕЛЬ ДВИГУНА

Математична модель авіаційного двигуна на основі диференціального рівняння. Параметри: ω, коефіцієнти передачі, стала часу.

📊

ВІЗУАЛІЗАЦІЯ

Автоматична побудова графіків засобами Python/matplotlib. Інтерактивне введення параметрів з валідацією через консоль.

✓

ТЕСТУВАННЯ

Повне покриття unit-тестами (24 тести). Використано Google Test Framework. Збірка через CMake.

📁

ТЕХНОЛОГІЧНИЙ СТЕК

C++17, CMake, Google Test, Python 3, matplotlib, CSV-формат для сумісності з Excel та LibreOffice.

🔬

ВЕРИФІКАЦІЯ

Порівняння результатів моделювання з аналітичним розв'язком. Контроль похибки на кожному кроці інтегрування.
